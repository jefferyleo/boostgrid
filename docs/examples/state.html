State persistence (localStorage)

With stateSave: true, Boostgrid mirrors the table's view — page, page-size, search, sort, column visibility, and (optionally) selection — to localStorage on every change. Reload the page or revisit later and the grid restores itself before the first paint.

Try this: search for "alpha", jump to page 2, hide a column via the toolbar dropdown, then refresh the browser. Everything snaps back.

Storage key: boostgrid:ex-state
ID Sender Subject Received 
const [grid] = attach("#grid", {
  stateSave: true,
  // Default key is `boostgrid:<table-id>` (or a column-id hash if no id)
  stateKey:  "my-grid",
});

// Wipe the persisted payload (does NOT reset the live grid)
grid.clearSavedState();

// To roll the live state back too:
grid.clearSavedState().reload();
// localStorage["boostgrid:my-grid"]
