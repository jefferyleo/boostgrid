Typed rows (TypeScript generics)

Boostgrid<TRow> threads your row interface through every row-shaped getter, input, and footer context — no casts at the call site. The grid below renders the same five rows, but the consuming code is fully typed against an explicit Message interface.

ID Sender Subject Received 
import { attach } from "boostgrid";

interface Message {
  id:       number;
  sender:   string;
  subject:  string;
  received: string;
}

const [grid] = attach<Message>("#grid");

// All row-shaped APIs are typed as Message[]:
const rows = grid.getCurrentRows();    // Message[]
const all  = grid.getAllRows();        // Message[]
const filt = grid.getFilteredRows();   // Message[]

// IntelliSense knows about every Message field — no casts:
const latest = rows[0].subject;        // string
grid.append([{ id: 99, sender: "x@y.z", subject: "Hi", received: "2026-01-01" }]);
